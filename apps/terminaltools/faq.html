Frequently Asked Questions



Which terminal applications are supported by Terminal Tools?

Terminal Tools currently supports the default macOS Terminal application and iTerm 2. Support for other applications may be added in the future!



What features do I get in the free version compared to Pro?

The free version includes native Finder integration for opening a new terminal window, global keyboard shortcut creation, and changing your terminal application. Pro is available as a one-time in-app purchase, and includes everything in the free version, along with the ability to create custom commands to run in the Finder and from the menu bar.



How do I enable the Finder toolbar button or right-click menu options?

Make sure that the Terminal Tools Finder extension is enabled under System Preferences > Extensions > Finder Extensions. If the extension is enabled, but the toolbar button is still not showing, right-click on the Finder toolbar, select "Customize Toolbar...", and drag the Terminal icon into the toolbar. For right-click menu options, make sure "Show right-click menu options" is enabled in Terminal Tools' preferences. You may also try relaunching the Finder by holding Option + Control and clicking the Finder icon in the dock.



Why aren't commands working in the menu bar or Finder?

Terminal Tools requires automation permissions to send custom commands to your terminal. In System Preferences, go to Security & Privacy > Privacy > Automation, and ensure that the checkboxes under Terminal Tools are enabled.



Why does an extra window get created when running a command while the terminal is not running?

This is a known issue when a macOS application tries to interact with another application that is not open. There is unforunately no solution at the moment.
